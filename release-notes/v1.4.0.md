# Release Notes — v1.4.0

**Released:** 2026-03-28

## What's New

### Signed Inheritance Bundles

All inheritance bundles are now signed with Ed25519. The child runtime verifies the signature
before applying any capability update. This prevents tampered or replayed bundles from being
applied, even if the distribution server is compromised.

Key changes:
- Every manifest now includes a `keyId` field identifying the signing key
- The `keyId` is covered by the signature, making key rotation tamper-proof
- Public key is distributed alongside the runtime and stored at `~/.greenfrog/public-key.pem`
- Standalone verification: `node tools/verify-release.js --manifest <path>`

### Improved Installer

The installer scripts now accept `--enrollment-url` as a command-line argument, eliminating
the need to manually edit configuration files after installation.

```bash
bash install.sh --enrollment-url "https://your-server.example.com/api/distribution/enroll"
```

### Child Status API

A new public (no-auth) endpoint returns the full operational state of a child instance:

```
GET /api/distribution/status
```

Returns enrollment state, inheritance version, credential presence, backflow queue depth,
and public key status.

## Compatibility

- **Runtime requirement:** Node.js >= 18
- **Compatibility constraint:** `>=1.0.0 <2.0.0`
- **Breaking changes:** None. v1.4.0 is backwards-compatible with v1.x child instances.

---

## Security

- `keys/mother-private.pem` is never included in any distribution artifact. Verified by
  automated security audit in the release pipeline.
- Ed25519 signatures use `null` algorithm parameter (Node.js `crypto.sign(null, ...)`), as
  required by the Ed25519 specification.
- Bundle hash (SHA-256) is verified independently of the signature.

---

## Upgrade

No manual action required. Enrolled child instances will receive this update automatically
on their next inheritance check cycle.

To force an immediate check:
```bash
curl -X POST http://localhost:18889/api/distribution/inheritance/check \
  -H "Authorization: Bearer <token>"
```
